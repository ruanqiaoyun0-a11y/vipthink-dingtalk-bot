
# VIPthink台湾市场课程顾问培训系统 - 实施计划

---

## 一、项目概述

根据PRD和技术架构文档，本项目需要实现以下核心功能：
1. **学习模块**：DAY1-DAY4知识点学习与闯关练习
2. **考核模块**：每日考核测试
3. **用户模块**：登录认证与学习记录
4. **钉钉集成**：机器人跳转与晚间播报

## 三、需要配置的Skill/工具

### 3.1 钉钉机器人配置

**配置步骤：**
1. 打开钉钉群 → 群设置 → 智能群助手 → 添加机器人 → 自定义机器人
2. 输入机器人名称（如：VIPthink培训助手）
3. 复制生成的Webhook地址
4. 配置安全设置：
   - IP白名单：添加服务器IP地址
   - 自定义关键词：添加"学习"、"播报"、"成绩"等

**配置项（需填写到.env文件）：**
```bash
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=xxx
DINGTALK_SECRET=xxx
```

### 3.2 数据库配置

**MongoDB配置：**
```bash
# 本地开发
MONGODB_URI=mongodb://localhost:27017/vipthink

# 生产环境（需配置用户名密码）
MONGODB_URI=mongodb://user:password@host:port/vipthink
```

### 3.3 JWT配置

```bash
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d
```

### 3.4 其他配置

```bash
PORT=3000
NODE_ENV=development
```

## 四、数据初始化

### 4.1 初始用户数据

**管理员账号（需手动创建）：**
```json
{
  "name": "管理员",
  "phone": "13800138000",
  "password": "bcrypt加密后的密码",
  "role": "admin"
}
```

**学员账号（可通过CSV导入或API创建）：**
```json
{
  "name": "学员姓名",
  "phone": "学员手机号",
  "password": "默认密码",
  "role": "student"
}
```

### 4.2 知识点数据结构

需要为DAY1-DAY4分别添加知识点，每个知识点包含：
- day: 天数（1-4）
- title: 知识点标题
- content: 知识点内容（销售话术要点）
- order: 排序序号

### 4.3 题目数据结构

需要为每天添加练习题目和考核题目：
- day: 天数（1-4）
- type: "practice" 或 "exam"
- question: 题目内容
- options: 选项数组（4个选项）
- answer: 正确答案索引（0-3）
- explanation: 答案解析

## 六、风险评估与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 钉钉机器人消息发送失败 | 中 | 中 | 增加重试机制，记录日志 |
| MongoDB连接失败 | 低 | 高 | 添加连接重试，健康检查 |
| JWT Token泄露 | 低 | 高 | 设置合理过期时间，HTTPS |
| 并发用户访问 | 低 | 中 | 考虑添加Redis缓存 |
| 数据丢失 | 低 | 高 | 定期备份数据库 |
