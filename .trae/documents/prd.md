
# VIPthink台湾市场课程顾问培训系统 - PRD文档

## 一、项目概述

### 1.1 项目背景
本项目旨在为VIPthink台湾市场课程顾问培训打造一个在线学习平台，通过闯关练习的形式帮助学员学习销售话术知识点，并进行考核评估。

### 1.2 目标用户
- **学员**：VIPthink台湾市场课程顾问
- **管理员**：培训师/管理人员

### 1.3 核心价值
- 系统化的知识点学习流程
- 互动式闯关练习体验
- 实时学习数据追踪
- 钉钉集成实现便捷学习和数据播报

---

## 二、功能需求

### 2.1 学习模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 学习首页 | 展示DAY1-DAY4学习进度卡片 | 高 |
| 知识点学习 | 按天展示知识点内容 | 高 |
| 闯关练习 | 选择题/情景模拟练习 | 高 |
| 考核测试 | 每日考核题目 | 高 |

### 2.2 用户模块

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 用户登录 | 钉钉扫码/账号密码登录 | 高 |
| 学习记录 | 记录练习次数和成绩 | 高 |
| 进度追踪 | 显示每日完成状态 | 高 |

### 2.3 钉钉集成

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 机器人跳转 | 点击钉钉消息跳转网页 | 高 |
| 晚间播报 | 21:30向管理群播报数据 | 高 |

### 2.4 管理后台

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 学员管理 | 查看学员列表和学习数据 | 中 |
| 内容管理 | 管理知识点和题目 | 中 |
| 数据统计 | 汇总分析学习数据 | 中 |

---

## 三、业务流程

### 3.1 学习流程
```
登录 → 选择天数 → 学习知识点 → 闯关练习 → 考核测试 → 查看成绩
```

### 3.2 数据播报流程
```
定时任务(21:30) → 统计数据 → 调用钉钉API → 发送到管理群
```

## 四、技术方案

### 4.1 前端技术栈
- React 18 + TypeScript
- TailwindCSS 3
- React Router
- Axios

### 4.2 后端技术栈
- Node.js + Express
- MongoDB
- 钉钉机器人SDK

### 4.3 关键API

| API路径 | 方法 | 功能 |
|---------|------|------|
| /api/users/login | POST | 用户登录 |
| /api/learning/day/:day | GET | 获取当天知识点 |
| /api/practice | POST | 提交练习答案 |
| /api/exam | POST | 提交考核答案 |
| /api/records | GET | 获取学习记录 |

## 五、页面结构

| 页面 | 路径 | 功能 |
|------|------|------|

## 六、数据模型

### 6.1 User（用户）
```
{
  id: string,
  name: string,
  phone: string,
  role: 'student' | 'admin',
  createdAt: Date
}
```

### 6.2 LearningRecord（学习记录）
```
{
  userId: string,
  day: number,
  practiceCount: number,
  examScore: number,
  completed: boolean,
  updatedAt: Date
}
```

### 6.3 KnowledgePoint（知识点）
```
{
  day: number,
  title: string,
  content: string,
  order: number
}
```

### 6.4 Question（题目）
```
{
  day: number,
  type: 'practice' | 'exam',
  question: string,
  options: string[],
  answer: number,
  explanation: string
}
```

## 七、进度规划

| 阶段 | 时间 | 任务 |
|------|------|------|
| 第一阶段 | 1-2天 | 项目初始化、基础架构搭建 |

| 第三阶段 | 5-6天 | 考核模块、成绩统计 |

## 八、需要配置的技能/Skill

### 8.1 钉钉机器人配置
- 创建钉钉自定义机器人
- 获取机器人Webhook地址
- 配置安全设置（IP白名单/自定义关键词）

### 8.2 环境变量配置
```
PORT=3000
MONGODB_URI=mongodb://localhost:27017/vipthink
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=xxx
DINGTALK_SECRET=xxx
```

### 8.3 数据库初始化
- 创建MongoDB数据库
- 初始化知识点和题目数据

## 九、风险与应对

| 风险点 | 应对措施 |
|--------|----------|
| 钉钉消息发送失败 | 增加重试机制和日志记录 |
| 学员数据丢失 | 定期备份数据库 |
| 并发访问压力 | 考虑增加Redis缓存 |
